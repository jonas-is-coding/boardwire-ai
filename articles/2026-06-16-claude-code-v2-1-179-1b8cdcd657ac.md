# Claude Code v2.1.179

Concrete bug fix release for Claude Code, improving stability and developer experience for users.

<h2>What's changed</h2>
<ul>
<li>Fixed mid-stream connection drops: partial responses are now preserved instead of showing a raw error, and the spinner no longer gets stuck at "running tool"</li>
<li>Fixed mouse-wheel scrolling in WSL2 under Windows Terminal and VS Code (regression in 2.1.172)</li>
<li>Fixed a sandbox <code>denyRead</code>/<code>allowRead</code> glob over a large directory tree making the Bash tool description enormous and the session unusable on Linux</li>
<li>Fixed the feedback survey capturing a single-digit reply as a session rating immediately after a turn completes</li>
<li>Fixed the welcome screen stacking multiple promotional banners — at most one promo now shows per session</li>
<li>Fixed Ctrl+O not showing the subagent's transcript when viewing a subagent</li>
<li>Fixed clicking the prompt input not returning focus from the subagent/footer panel</li>
<li>Fixed remote session background tasks appearing stuck as "still running" between turns</li>
<li>Improved plugin loading performance in remote sessions</li>
</ul>

This story surfaced via Claude Code Releases. For the original details and any numbers we have not confirmed here, follow the source below.
