# Claude Code v2.1.181

New CLI version with concrete features for dynamic control, macOS integration, and runtime upgrade.

<h2>What's changed</h2>
<ul>
<li>Added <code>/config key=value</code> syntax to set any setting from the prompt (e.g. <code>/config thinking=false</code>) — works in interactive, <code>-p</code>, and Remote Control</li>
<li>Added <code>sandbox.allowAppleEvents</code> opt-in setting that lets sandboxed commands send Apple Events on macOS</li>
<li>Added <code>CLAUDE_CLIENT_PRESENCE_FILE</code> environment variable: point it at a marker file to suppress mobile push notifications while you're at the machine</li>
<li>Upgraded the bundled Bun runtime to 1.4</li>
<li>Improved streaming of long paragraphs: text now appears line-by-line instead of waiting for the first line break</li>
<li>Improved auto-retry: API connection drops mid-thinking now automatically retry instead of showing "Connection closed while thinking"</li>
<li>Improved the subagent panel: idle subagents auto-hide after 30s, the list caps at 5 rows with scroll hints, and keyboard hints now show in the footer</li>
<li>Improved the MCP OAuth browser page to match Claude Code's visual style and auto-close on success</li>
<li>Changed fullscreen mode URL opening to require Cmd+click (macOS) / Ctrl+click, matching native terminal behavior</li>
<li>Changed the <code>Improved N memories</code> line to no longer list individual files outside verbose mode</li>
<li>Fixed prompt caching not reading on custom <code>ANTHROPIC_BASE_URL</code> and on Foundry due to a per-request attestation token changing every turn</li>
<li>Fixed Write/Edit producing 0-byte or truncated files on network drives and cloud-synced folders</li>
<li>Fixed <code>open</code>, <code>osascript</code>, and browser-based auth flows failing with error -600 on macOS by adding the Apple Events entitlement</li>
<li>Fixed a startup regression (~120ms per launch in fresh environments, introduced in 2.1.169): the first prompt no longer waits for the managed-settings fetch when no MCP servers are configured</li>
<li>Fixed startup blocking with a blank terminal for up to 15 seconds when the account settings fetch is slow on a degraded network</li>
<li>Fixed startup crash (<code>TypeError: Cannot read properties of null</code>) when <code>.claude.json</code> contains corrupted null project entries</li>
<li>Fixed macOS TUI freezing at session start (Ctrl+C unresponsive) when Spotlight is busy reindexing</li>
<li>Fixed long-running idle sessions losing their history when another Claude Code process ran the 30-day transcript cleanup</li>
<li>Fixed foreground subagents spawning unbounded nested chains; they now respect the same 5-level depth limit as background subagents</li>
<li>Fixed <code>/recap</code> and conversation forks using the previous model immediately after a model switch</li>
<li>Fixed subagent "Thinking" duration showing the parent agent's elapsed time instead of the subagent's own</li>
<li>Fixed subagents blocked on a nested agent showing a ticking elapsed time instead of "waiting" in the agent panel</li>
<li>Fixed the API retry indicator ("Retrying in 0s · attempt N/10") staying on screen after the retry succeeded</li>
<li>Fixed AWS <code>awsCredentialExport</code> credentials with a short remaining lifetime causing credential refreshes every minute, and now accepts the JSON shape from <code>aws configure export-credentials</code></li>
<li>Fixed <code>claude mcp get</code>/<code>list</code> showing <code>✓ Connected</code> when tools/list fails; they now show <code>! Connected · tools fetch failed</code> with the error detail</li>
<li>Fixed <code>/remote-control</code> leaving a stale "connecting…" line; it now confirms in the transcript once connected</li>
<li>Fixed ExitWorktree refusing to remove a clean worktree with "Could not verify worktree state" when bare <code>git</code> cannot be resolved on Windows</li>
<li>Fixed settings changes (such as <code>/effort</code> or <code>/model</code>) failing with ENOENT when <code>~/.claude/settings.json</code> is a relative symlink under a symlinked <code>~/.claude</code></li>
<li>Fixed IDE selection line numbers in context reminders being off by one (IntelliJ and VS Code)</li>
<li>Fixed Ctrl+C in fullscreen after a native terminal selection (modifier+drag) overwriting the clipboard with the app's prior selection</li>
<li>Fixed Ctrl+V showing "No image found in clipboard" instead of pasting when the clipboard contains text</li>
<li>Fixed agent creation failing with "EEXIST: file already exists" when the agents directory already exists (Windows/OneDrive)</li>
<li>Fixed AskUserQuestion preview content being cut off at the dialog edge instead of word-wrapping</li>
<li>Fixed AskUserQuestion multi-select questions silently dropping a typed "Other" free-text answer when submitting</li>
<li>Fixed <code>/stats</code> "Most active day" and daily token chart dates showing one day early in UTC-negative timezones</li>
<li>Fixed <code>/copy</code> and copy-on-select on Linux not detecting a clipboard utility installed after Claude Code started</li>
<li>Fixed tab-indented code rendering with incorrect indentation in the Write (create-file) preview</li>
<li>Fixed user prompts queued mid-turn not showing a full-width background highlight in the transcript</li>
<li>Fixed the activity spinner's pulse dwelling on the wrong glyph size in Ghostty</li>
</ul>

This story surfaced via Claude Code Releases. For the original details and any numbers we have not confirmed here, follow the source below.
